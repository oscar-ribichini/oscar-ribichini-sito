3 aggiunte per rendere il metodo più robusto
1
Soglia di attivazione della Fase 0
Definire in 2 righe quando si attiva e quando no. Esempio: "Si attiva per qualsiasi task che richiede più di un file o più di una sessione. Non si attiva per domande rapide eseguibili in un solo messaggio."
2
Trigger automatico di controllo
Una riga fissa nel file _STATO_ che Oscar vede all'inizio di ogni sessione: "Stai seguendo il metodo? Fase 0 completata?" — rende il metodo autoattivante invece di dipendere dalla memoria.
3
Gestione sessione interrotta
Se la sessione finisce senza aggiornamento di _STATO_, il primo messaggio della sessione successiva è: /stallo — così il sistema si riallinea automaticamente.
