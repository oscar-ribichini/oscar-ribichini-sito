Come funziona — passo per passo
1
Oscar scrive il task — Claude non risponde subito
Qualunque cosa scriva Oscar — un documento, un'analisi, un'email, un piano — Claude non produce output. Si ferma e legge tutto il contesto disponibile: CLAUDE.md, memoria, file _STATO_ del progetto se esiste.
2
Claude valuta le 18 lenti — assegna semaforo
Per ogni lente rilevante Claude assegna uno stato. Non le usa tutte — sceglie quelle che impattano davvero questo task specifico.
🟢 VerdeHo informazioni sufficienti — posso procedere
🟡 GialloHo info parziali — posso procedere con assunzione dichiarata
🔴 RossoManca qualcosa di critico — mi fermo e chiedo
3
Claude fa le domande — costruite sulle lenti rosse
Almeno 3 domande, massimo 20, in blocchi da 5–7. Ogni domanda nasce da una lente rossa o gialla — non sono domande generiche, sono domande che cambiano l'output. Ogni 5–7 domande: checkpoint visivo con stato lenti aggiornato.
→Una domanda alla volta — mai tutte insieme
→Ogni domanda è collegata a una lente specifica
→Oscar può correggere ad ogni checkpoint
→Claude dichiara le assunzioni sulle lenti gialle
4
Regola di stop — Claude valuta se può produrre
Claude non smette di fare domande "quando sente di sapere abbastanza" — smette quando supera un test concreto e oggettivo.
Test di stop: "Riesco a scrivere le prime 3 sezioni dell'output senza scrivere [DA VERIFICARE] in nessun punto?"
Sì → le domande sono finite. Claude dichiara le assunzioni rimaste (lenti gialle) e produce.
No → Claude fa un'altra domanda mirata sulla sezione bloccata.
5
Claude produce — con assunzioni dichiarate
Output pulito, senza [DA VERIFICARE]. Prima dell'output Claude scrive una riga: "Assunzioni dichiarate: [lista lenti gialle]" — Oscar sa esattamente cosa resta da verificare.
6
Fine sessione — Claude aggiorna _STATO_ automaticamente
Senza aspettare che Oscar lo chieda. Il file _STATO_ viene aggiornato con: dove siamo, prossima azione, blocchi aperti, file attivi, scadenze. Prima riga fissa:
⚙️ METODO APPLICATO — Fase 0 completata: Sì  |  Data: [oggi]  |  Lenti rosse risolte: [n]
↺
Sessione interrotta o confusione — /stallo
Se la sessione finisce male o Oscar si sente perso: il primo messaggio della sessione successiva è /stallo. Claude legge in silenzio, fa lo specchio, analizza le lenti, propone 3 domande + raccomandazione. Oscar decide.

La disciplina sta in Claude — non in Oscar.

Oscar non deve ricordare di attivare nulla. Claude parte sempre con le domande. È il suo modo di lavorare in questo workspace — non un comando, non una fase speciale. È il default.

Può sempre accelerare Se Oscar scrive "ho fretta, vai" Claude riduce le domande al minimo indispensabile (solo lenti rosse) e dichiara tutte le assunzioni.
